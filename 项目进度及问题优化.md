### 证书姓名乱码修复
1. **根因**：Windows 终端 curl 默认用 GBK 编码发送中文，服务器收到乱码
   - 用 `printf` 写临时文件 + `curl -d @file` 确保 UTF-8
   - `readBody` 也改为 `Buffer.concat(chunks).toString("utf8")` 更安全
2. **服务器缺字体**：阿里普惠体 OTF 文件未上传到服务器，已补传
3. **SKILL.md 更新**：curl 命令改为文件发送方式，性别必须确认不能默认

### 踩坑记录
- **Windows curl 中文编码**：`-d '{"name":"晓峰"}'` 发的是 GBK 字节，不是 UTF-8。解决：`printf '...' > /tmp/file.json && curl -d @/tmp/file.json`
- **调试方法**：直接在服务器用 `node -e` 调 cert-gen 函数可跳过 HTTP 层快速验证渲染

## 2026-04-26 安装器 v1.2.2 — 升级 openclaw 2026.4.23 + 移除微信（已完成）

### 变更内容
1. **OpenClaw 版本升级**: 2026.4.15 → 2026.4.23
   - Gateway RPC 层修复（`openclaw status`/`channels list` 不再挂死）
   - 启动速度优化（从 228s 降到 7-15s）
2. **移除微信通道功能**
   - 原因：openclaw-weixin 插件在 2026.4.23 上 channel 注册失败（`OFF / not configured`）
   - 插件加载正常但 gateway 不调用 `startAccount`，属于插件与 gateway 兼容性问题
   - 移除范围：main.js（IPC handlers + stopGateway/restartGateway）、preload.js、config.js、renderer.js（164行扫码流程）、index.html（tab + dashboard 卡片）
   - scripts/wechat-scan.js 保留（未来可能恢复）
3. **安装器版本**: 1.2.1 → 1.2.2
4. **防止 gateway 不必要重启**（追加修复）
   - 问题：每次启动都写 token/port 到 openclaw.json → gateway 检测到 config change → SIGUSR1 重启 → WebSocket 1006 断连 + CMD 弹窗
   - 修复：三处 writeConfig 改为变化时才写入（`if (old !== new)`）
5. **重启按钮只停不启**（追加修复）
   - 问题：`gateway-restart` handler 用 `ipcMain.invoke` 调 `launch-openclaw`，在 main 进程内部无效
   - 修复：提取 `doLaunchOpenclaw()` 函数，handler 直接调用
6. **飞书 EPERM 循环重启**（追加修复）
   - 问题：gateway 加载 feishu 时 `.openclaw-install-stage` EPERM → 标记失败 → config change → SIGUSR1 循环重启 → CMD 弹窗
   - 修复：启动前清理 install-stage 残留文件；重启按钮简化为直接调 doLaunchOpenclaw（删除重复停止逻辑）
7. **飞书保存按钮无反馈**
   - 修复：添加 loading 状态、空值校验、成功提示
   - 删除 saveChannelConfig 中无用的 execSync CLI 同步（可能卡 15 秒）

### 构建产物

- `dist/OpenClaw-Portable-v1.2.2.exe` (304MB, 便携版)

### 升级流程
- 安装器已内置升级检测（detect.js 版本对比 → renderer.js 升级 UI）
- `npm install -g openclaw@新版本` 自动覆盖旧版本，保留配置

### 已知问题（已接受，后续解决）
- **CMD 窗口闪现**: 启动/重启 gateway 时仍有短暂 CMD 窗口闪现（约 1 秒）。根因：`shell: true` 必须使用（否则进程链断裂 gateway 立即死亡），但 `windowsHide` 不可靠。尝试了 10+ 种方案（PowerShell `-WindowStyle Hidden`、`cmd /c`、`detached`、各种组合），要么弹窗依旧，要么 gateway 无法启动。最终回退到 `shell: true + windowsHide`（commit `6765967`），功能完全正常但有闪现。
  - 可能方案：wscript shell、Windows Service、Scheduled Task
- **openclaw-weixin 兼容性**: 等待插件更新修复 channel 注册问题，再恢复微信功能（代码在 git 历史中）

### 当前稳定版本
- **commit**: `6765967` — `fix: 回退 spawnHidden/execHidden 为 shell:true + execSync 方案`
- **功能状态**: 全部正常（启动/停止/重启/飞书通道/模型配置/激活码）
- **spawnHidden**: `shell: true + windowsHide: true`（有 CMD 闪现但稳定）
- **execHidden**: `execSync + windowsHide: true`（短命令可靠）
